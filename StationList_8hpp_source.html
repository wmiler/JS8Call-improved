16// Class StationList
17//
18// Encapsulates information about a collection of unique operating
19// stations per band. The implementation is a table model with the
20// first column being the unique (within the table rows) band name
21// and, the second the frequency offset for transverter usage and,
22// the third the antenna description. All are editable.
23//
24// Responsibilities
25//
26// Stores internally an unordered table of bands.
27//
28// If an ordered representaion is required then wrapping with an
29// appropriate proxy model is sufficient
30// e.g. QSortFilterProxyModel. A custom SortRole role is provided for
31// the band name column which returns a numeric value (Bands lower
32// frequency limit) which gives a strict frequency ordering by band.
33//
34// Collaborations
35//
36// Implements the QAbstractTableModel interface for a grid of bands
